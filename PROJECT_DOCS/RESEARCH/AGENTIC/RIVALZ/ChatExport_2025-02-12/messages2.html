-Research Sum Up - Roadmap

After researching a bit around Agents, how they are triggered, what differentiates them from AI Apps; and checking on a few competitors or similar platforms Documentations (i.e. VoiceFlow, LangChain) this are our findings:

-AI Apps need Human input, and interaction to work.

-To qualify as an AI Agent, not only no Human interaction is required but:
-Has the ability to take action over external systems
-Has the ability to decide when and how to produce this actions
-Has the ability to write and read, logs of previous inputs and outcomes/outputs, to decide if other different actions has to be taken
- Is persistent
- May or may not work with Async Flows.
‚Äî‚Äî‚Äî‚Äî‚Äî‚Äî‚Äî‚Äî‚Äî‚Äî‚Äî‚Äî‚Äî‚Äî‚Äî‚Äî-
When it comes to AI App functionalities, we have mostly the same as our competitors, with a few exceptions:
-They have the capability to take inputs from users, from other platforms (i.e. Whatsapp, Telegram)
-Chatbot produces outcome on other platforms. (I.e. Telegram, Whatsapp, Twitter, Instagram, etc)
-capability of calling other flows that were previously created.

example use cases:
-Send a text message to the company whatsapp, the chatbot search for an answer in their internal docs, and answer back to the user on Whatsapp

-User ask the chatbot to create a post about certain subject related (or not) to the company. The chatbot posts it on several socials, according to their respective format

-User sends a message through Telegram Whatsapp or even the chatbot interface, asking to create a quote for a job, and register it in the company's Quotes Spreadsheet. The chatbot produces the Quote in an specific format, and connects to the spread sheet to produce an entry there.

‚Äî‚Äî‚Äî‚Äî‚Äî‚Äî‚Äî‚Äî‚Äî‚Äî‚Äî‚Äî‚Äî‚Äî‚Äî‚Äî-
About AI Agents we need to implement, we don't have Agents functionality.

-We need to implement APIs that directly and automatically, receive info from external systems.
-The workflow should take this as input, in the same fashion it does when a user inputs text (this is what triggers, the agent)
-The agents needs to analyze the received info, compare it to their RAG, and produce and Output.
-The Output can produce an action over an external system. API or similar connection, with a predefine prompt on how the output should be structured for the external system to take it as an order/command
- The Agent needs to record a log of inputs and outcomes, for future analysis, for future decision making, and for real time training.

Flow should look something like this:
External DATA input‚Äî->Agent compares values to RAG + logs ‚Äî-> Agent performs action on external system/Agent doesn't perform action‚Äî->Agent logs process

Example Use cases:

-Machinery with various sensors connected through APIs to Agent. It produces Data at regular intervals (or real time). The agent is connected through APIs (or similar) to it, receiving the data produced by the sensors. Compares the data received with the RAG it has, and logs of previous instances. It detects same off values, produces an email/text alert, and/or sends a stop command through APIs to machinery.

-Store inventory system connected to agent. In regular intervals (or real time) it sends stock related data to Agent. Agent checks DATA, compares it to RAG + Logs, and can analyses if Restocking is needed, predict items exhaustion time, and paces a purchase order, based on delivery time, outflow of inventory (logs) etc etc. Or just emits an alert via Email



‚Äî‚Äî‚Äî‚Äî‚Äî‚Äî‚Äî‚Äî‚Äî‚Äî-
Conclusion
At the moment, our platform is capable of producing AI Apps, that only output answers through the chatbot, with no possibility of integrating other platforms or systems for more elaborated outcomes.

The objective should be to be able to build AI Agents that can interact and action over different external systems and platforms. Autonomy will be achieved, once we can connect to other systems that produce DATA. this Data is what triggers the Agents to initiate a workflow.
